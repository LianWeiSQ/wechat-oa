# 微信公众号 + 小红书运营总纲

更新日期：2026-05-20

## 账号定位

把 AI Agent 从“模型热闹”拉回“工程上线”。账号要长期占住一个清晰心智：

> 面向技术负责人、AI 工程师、创业团队和企业 AI 采购方，拆解 Agent 真正落地时的状态管理、工具编排、可靠性、成本和组织协作问题。

公众号承担“深度信任”和“体系化内容资产”，小红书承担“发现流增长”和“轻量观点种草”。两边不是重复搬运，而是同一主题的不同颗粒度表达。

## 目标读者

1. AI Agent / AI Infra 工程师：关心框架选型、状态恢复、工具调用、评测、生产稳定性。
2. 技术负责人 / CTO：关心 Agent 项目为什么难上线、怎么分阶段投入、团队如何组织。
3. 企业老板 / AI 咨询客户：关心 AI 项目是否能真正降本增效，如何判断供应商是不是只会讲模型。
4. 想转 Agent 方向的技术人：关心学习路径、项目作品、面试表达和工程判断力。

## 双平台分工

| 平台 | 核心任务 | 内容节奏 | 适合内容 |
| --- | --- | --- | --- |
| 微信公众号 | 建立专业信任，沉淀长文资产，承接咨询/合作/产品转化 | 每周 2 篇，稳定后加到 3 篇 | 深度拆解、框架源码解读、案例复盘、方法论、系列文章 |
| 小红书 | 获取新读者，测试选题，放大观点，收集评论反馈 | 每周 4 篇，热点期可 5 篇 | 观点卡片、清单、踩坑复盘、职业建议、评论区问答 |

## 内容比例

公众号采用 60/30/10：

- 60% 价值内容：工程拆解、架构取舍、源码阅读、方法论。
- 30% 互动内容：读者问题、投票、评论精选、案例征集。
- 10% 转化内容：咨询服务、工具产品、资料包、合作入口。

小红书采用 70/20/10：

- 70% 生活化/场景化内容：真实项目观察、工程师日常判断、上线踩坑。
- 20% 趋势参与：Agent 热点、模型发布、框架更新、行业争议。
- 10% 品牌直给：公众号长文引流、工具展示、服务介绍。

## 内容支柱

1. Agent 工程化：Harness、状态管理、工具调用、断点续跑、可靠性。
2. AI Infra 与架构：向量库、RAG、评测、观测、队列、权限、部署。
3. 框架源码拆解：Claude Code、LangGraph、DeerFlow、MCP、Browser Use 等。
4. 技术团队决策：模型选型、预算、项目边界、失败复盘、组织分工。
5. 技术人成长：Agent 面试、作品集、学习路线、从后端转 AI Infra。

## 周节奏

| 星期 | 公众号 | 小红书 | 运营动作 |
| --- | --- | --- | --- |
| 周一 | 选题会，确定本周深度文 | 发布观点型笔记 | 看上周数据，挑出可放大的评论 |
| 周二 | 写/改第一篇长文 | 评论区互动，私信沉淀问题 | 整理读者问题进入选题池 |
| 周三 | 不发或发短消息 | 发布清单型笔记 | 测试标题、封面关键词 |
| 周四 | 发布深度长文 | 同主题拆 1 条小红书 | 2 小时内高频回复评论 |
| 周五 | 准备周末文 | 发布踩坑/职业向笔记 | 观察热点，决定是否追题 |
| 周六 | 选发案例/问答文 | 轻量互动或停更 | 复盘私信和收藏率 |
| 周日 | 发布周总结或系列文 | 不强制发布 | 更新下周排期 |

| --- | --- |

## 小红书视觉与文案规则

- 封面优先 9:16，主标题 8-14 个字，副标题补充场景。
- 视觉保持“技术白板 + 工程杂志”风格：干净背景、低饱和色、少装饰、重点词高亮。
- 每篇正文控制在 450-900 字，段落短，第一屏必须有冲突。
- 每篇只讲一个判断，不把公众号长文全部塞进去。
- CTA 优先“收藏/评论关键词/去公众号看长文”，少用强销售表达。

## 固定转化路径

小红书笔记 -> 评论区关键词 -> 私信/主页引导 -> 公众号长文 -> 自动回复资料 -> 合作或产品入口。

公众号长文 -> 文末问题 -> 关键词自动回复 -> 读者问题池 -> 下一篇文章/小红书短内容。

## 核心 KPI

公众号：

- 打开率目标：30%+
- 文内点击率：5%+
- 完读率：50%+
- 月自然增长：10%-20%
- 取关率：低于 5%
- 菜单周点击率：20%+

小红书：

- 互动率：5%+
- 收藏率：8%+
- 分享率：2%+
- 评论中有效问题占比：30%+
- 月自然增长：15%-25%
- 每月至少 1 条笔记进入明显破圈区间

## 每周复盘模板

每周日记录一次：

1. 本周最强选题：标题、平台、阅读/曝光、收藏、评论。
2. 最强评论：读者真正关心什么。
3. 最弱内容：为什么弱，是标题、封面、选题还是表达颗粒度。
4. 下周放大动作：把哪条小红书扩成公众号，或把哪篇公众号拆成 3 条小红书。
5. 转化观察：关键词回复、私信、合作咨询、产品点击。

## 内容边界

- 不编造客户案例，不虚构“朋友项目”细节。
- 不用“普通人靠 Agent 逆袭”这类泛流量话术。
- 不承诺模型能力能直接替代团队。
- 不把源码拆解写成洗稿；必须有自己的工程判断。
- 不频繁硬卖课、硬拉群，先用内容建立信任。
