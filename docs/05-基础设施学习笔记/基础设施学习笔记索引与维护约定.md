# 基础设施学习笔记索引与维护约定

## 目的

这个目录用于维护 William 的基础设施学习笔记。后续你问我的 infra、后端、运行时、调度、存储、队列、缓存、可观测性、部署、数据库、分布式系统相关问题，我会优先把产出整理成文档落到这里，而不是只在对话里回答一次。

目标不是做零散问答，而是逐步积累成一套能复习、能串起来、能回头补充的学习资料。

## 使用规则

1. 每次一个问题，优先沉淀成一篇独立文档。
2. 文件名使用中文主题，不带日期，便于长期维护和回看。
3. 一篇文档尽量回答一个核心问题，不把多个大问题硬塞在一起。
4. 如果一个问题会引出多个子问题，先写主文，再在文末列“下一步可展开”。
5. 如果后续对同一主题有补充，优先更新原文档，再决定是否拆新文。

## 推荐结构

后续每篇 infra 学习笔记尽量按下面结构写：

1. 问题
2. 结论先行
3. 核心概念
4. 工作机制
5. 常见误区
6. 和当前项目的关系
7. 进一步学习方向

## 和当前项目的关系

这个仓库本身已经覆盖了不少适合做 infra 学习素材的内容：

- SQLite 与 Supabase 双存储路径
- Scheduled generation runtime
- Draft 持久化
- Writing agent 多阶段流水线
- Settings 和模型路由
- 失败重试与任务状态

所以这里的学习笔记，最好尽量和项目里的真实实现挂钩。这样你学的不是抽象概念，而是能落回当前代码和系统设计里的东西。
