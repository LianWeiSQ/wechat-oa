# 文档中心

这个目录按“先理解产品，再看运营，再看功能设计，最后看实施细节”的顺序维护。新文档优先放进中文编号目录，避免再出现一堆英文文件名堆在一起。

## 阅读顺序

1. [产品战略 / PRD](./01-产品战略/2026-05-22-微信公众号AI内容生产系统PRD战略分析.md)
   先看这里。它回答产品定位、竞品、核心用户、路线图和优先级。

2. [运营总纲](./02-运营策略/2026-05-20-微信公众号小红书运营总纲.md)
   看账号定位、目标读者、内容支柱和公众号/小红书分工。

3. [技术文章库与微信公众号工作台拆分](./03-功能设计/2026-05-15-技术文章库与微信公众号工作台拆分.md)
   看产品主界面的功能边界：知识库负责文章资产，公众号工作台负责生成和发布。

4. [前端编辑台视觉整改](./03-功能设计/2026-05-15-前端编辑台视觉整改.md)
   看当前前端视觉方向和已有改动。

5. [专业公众号长文生成设计](./03-功能设计/2026-05-14-专业公众号长文生成设计.md)
   看专业长文、配图、草稿生成链路的功能设计。

6. [实施计划](./04-实施计划/2026-05-14-专业公众号长文生成实施计划.md)
   只有要改代码或追历史实现细节时再看。

## 目录说明

| 目录 | 用途 | 什么时候看 |
| --- | --- | --- |
| `01-产品战略` | PRD、战略分析、产品方向、路线图 | 讨论产品该做什么、先做什么 |
| `02-运营策略` | 公众号和小红书运营、内容排期、发布清单 | 讨论账号定位、内容节奏、增长 |

| `90-历史归档` | 与当前主产品不直接相关的旧资料 | 需要查历史时 |

## 新文档命名规则

- 产品战略：`docs/01-产品战略/YYYY-MM-DD-中文标题.md`
- 运营策略：`docs/02-运营策略/YYYY-MM-DD-中文标题.md`
- 功能设计：`docs/03-功能设计/YYYY-MM-DD-中文标题.md`
- 实施计划：`docs/04-实施计划/YYYY-MM-DD-中文标题.md`
- 归档资料：`docs/90-历史归档/主题/中文标题.md`

文档标题和文件名尽量使用中文。英文技术名可以保留，例如 `Supabase`、`Postgres`、`Agent`、`PRD`。

## 当前主线

当前产品主线是“技术知识资产驱动的公众号生产系统”：

- 知识库：导入、阅读、收藏、分类、检索、管理技术文章。
- 结构资产：拆解文章写法，把好文章变成可复用结构。
- 写作工作台：从选题、参考文章、蓝图到原创草稿。
- 草稿箱：编辑、预览、配图、导出、推送微信草稿。

后续新增文档要能落到这条主线上。如果只是临时会话记录，不放进 `docs/` 主目录；完成沉淀后再整理到上面的四个主线目录或历史归档。
