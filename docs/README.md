# 文档中心

这个目录按“先理解产品，再看运营，再看功能设计，最后看实施细节”的顺序维护。新文档优先放进中文编号目录，避免再出现一堆英文文件名堆在一起。

## 阅读顺序

1. [产品战略 / PRD](./01-产品战略/README.md)
   先看这里。它回答产品定位、竞品、核心用户、路线图和优先级。

2. [运营策略](./02-运营策略/README.md)
   看账号定位、目标读者、内容支柱和公众号/小红书分工。

3. [功能设计](./03-功能设计/README.md)
   看产品主界面的功能边界：知识库负责文章资产，公众号工作台负责生成和发布。

4. [实施计划](./04-实施计划/README.md)
   只有要改代码或追历史实现细节时再看。

5. [基础设施学习笔记](./05-基础设施学习笔记/README.md)
   看基础设施学习索引、主题拆分和后续问题沉淀方式。

6. [历史归档](./90-历史归档/README.md)
   查个人站点草稿和其他已经退到侧线的资料。

## 新文档命名规则

- 产品战略：`docs/01-产品战略/中文标题.md`
- 运营策略：`docs/02-运营策略/中文标题.md`
- 功能设计：`docs/03-功能设计/中文标题.md`
- 实施计划：`docs/04-实施计划/中文标题.md`
- 基础设施学习笔记：`docs/05-基础设施学习笔记/中文标题.md`
- 归档资料：`docs/90-历史归档/主题/中文标题.md`

文档标题和文件名尽量使用中文。英文技术名可以保留，例如 `Supabase`、`Postgres`、`Agent`、`PRD`。

## 当前主线

当前产品主线是“技术知识资产驱动的公众号生产系统”：

- 知识库：导入、阅读、收藏、分类、检索、管理技术文章。
- 结构资产：拆解文章写法，把好文章变成可复用结构。
- 写作工作台：从选题、参考文章、蓝图到原创草稿。
- 草稿箱：编辑、预览、配图、导出、推送微信草稿。

后续新增文档要能落到这条主线上。如果只是临时会话记录，不放进 `docs/` 主目录；完成沉淀后再整理到上面的四个主线目录或历史归档。
