# AI 技术栈观察清单

适用场景：看到新模型、新框架、新 Agent Demo、AI 产品融资或企业落地案例时，用这份清单判断它真正改变了什么。
关键词建议：`技术栈`

## 1. 新模型

1. 它提升的是推理、多模态、长上下文、速度、成本，还是部署方式？
2. 它会降低哪个具体应用场景的成本？
3. 它让哪些原本不可用的工作流变得可用？
4. 它对 RAG、Agent、工具调用有什么直接影响？
5. 它是通用能力提升，还是打开了某个垂直场景？
6. 它的优势是否能被产品稳定利用，而不是只停留在 benchmark？

## 2. RAG / 知识库

1. 资料来源是否清楚？
2. 检索是否可评估？
3. 答案是否能引用来源？
4. 权限、更新、冲突和过期资料如何处理？
5. 这套知识库解决的是搜索问题、问答问题，还是业务流程问题？
6. 它是否减少了人的查找成本，还是只是换了一个聊天入口？

## 3. Agent / Harness

1. Agent 是否真的调用了外部工具？
2. 工具调用结果是否进入状态？
3. 执行中断后能否恢复？
4. 是否有权限边界和人工接管？
5. Demo 里的任务是否能稳定重复执行？
6. 它解决的是“会演示”，还是“能进入生产”？

## 4. AI 产品

1. 产品解决的是一个真实流程，还是只加了聊天框？
2. 用户为什么会持续使用？
3. 上下文从哪里来？
4. 输出如何进入下一步工作？
5. 产品的护城河来自模型、数据、工作流，还是分发渠道？
6. 它的失败场景是否被设计过？

## 5. 企业落地

1. 场景是否足够具体？
2. 原流程是否被改变？
3. 业务指标是否定义清楚？
4. 谁负责 AI 输出质量？
5. 数据和权限是否可控？
6. 如果模型能力变化，这个项目是否还能持续迭代？

## 6. 最终判断

看到一条 AI 新闻后，最后只问 3 件事：

1. 它改变了技术栈的哪一层？
2. 它会让哪些应用更便宜、更稳定或更容易上线？
3. 它是短期热闹，还是长期结构性变化？
