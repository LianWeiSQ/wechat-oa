# 别再问模型了

封面文案：

- 主标题：别再问模型了
- 副标题：Agent 真正卡在 Harness

正文：

很多团队做 Agent，一上来就问：

“用 GPT 还是 Claude？”
“要不要换最新模型？”
“长上下文是不是能解决问题？”

但真到上线时，最先出问题的往往不是模型。

我最近复盘过的一个典型场景是：测试环境里效果很好，Demo 也很惊艳。可一到真实用户长任务，网络抖动一次、工具调用超时一次，整个会话就挂了。

模型其实没变。

真正没准备好的是 Harness。

可以把 Harness 理解成 Agent 的运行时骨架：它负责入口适配、会话成型、状态管理、工具编排、执行循环和失败恢复。

模型负责“想下一步做什么”，Harness 负责“这一步能不能被可靠执行，断了能不能续，副作用会不会重复发生”。

一个反常识判断：

模型决定 Agent 的能力上限，但 Harness 决定你能不能稳定触达那个上限。

如果你的 Agent 还回答不清这几个问题，就别急着换模型：

1. 状态存在哪里？
2. 每一步什么时候持久化？
3. 工具调用失败后怎么恢复？
4. 已经成功的外部动作如何避免重复执行？
5. 长任务中断后能从哪里继续？

模型越强，执行链路越长，中间状态越多，Harness 反而越重要。

所以做 Agent 工程，不要只盯着模型榜单。先把状态管理、工具编排和恢复闭环做扎实，这才是从 Demo 走向生产的分水岭。
